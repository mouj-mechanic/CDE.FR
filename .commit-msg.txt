feat(provider): add OpenAI GPT Image as primary try-on provider

- New lib/providers/openaiImage.ts using the official openai SDK with
  multi-image edit (gpt-image-1). Reads OPENAI_API_KEY server-side only.
- Server-side mask conversion (B/W -> alpha PNG) via sharp so our existing
  watch contact-band mask works unchanged with OpenAI's transparent-mask
  convention.
- buildOpenAIPrompt() in lib/providers/prompts.ts strict prompts for
  watch / glasses / hand-jewelry / headwear / clothes following the spec.
- tryOnService now supports AI_TRYON_PROVIDER=openai and "auto" prefers
  OpenAI when OPENAI_API_KEY is present, fal.ai otherwise.
- /api/try-on returns renderMode=gpt-image-edit + debug.usedOpenAI /
  usedFal / maskUsed for downstream UI and analytics.
- /api/ai-status reports hasOpenAIKey and the active mode.
- ResultView shows a "Generated with GPT Image" badge and hides the
  fal.ai provider line when provider=openai.
- .env.example + README document OPENAI_API_KEY, OPENAI_IMAGE_MODEL,
  OPENAI_IMAGE_SIZE, OPENAI_IMAGE_QUALITY, OPENAI_USE_MASKED_EDIT.

Note: defaults to gpt-image-1 (the model that actually exists today).
Override with OPENAI_IMAGE_MODEL=gpt-image-2 once it ships, no code
change needed.
